## 1. Fix Gap-Down/Gap-Up Entry Zone

**Problem**: Entry zones based on prev_high miss gap-down opens
- 1303.SR Jun 1: Entry 16.75, Open 16.31 (gapped DOWN 2.6%), never hit
- Screener assumes bullish continuation but market gaps down

**Current Code** (screener.py):
```python
if close >= prev_high * 0.99:
    entry_low = round(prev_high * 0.995, 2)  # ← Too high
    entry_high = round(close * 1.01, 2)
```

**Suggested Fix**:
```python
# v4.1: Use lower of prev_high*0.995 or close*0.98
if close >= prev_high * 0.99:
    entry_low = round(min(prev_high * 0.995, close * 0.98), 2)
    entry_high = round(close * 1.01, 2)
else:
    entry_low = round(prev_low * 0.998, 2)
    entry_high = round(prev_high * 1.002, 2)
```

**Test Result**: entry_low = 16.41 vs open 16.31 → STILL doesn't hit!
Need even wider: `close * 0.95` = 15.91 would hit.

## 5. Address Gap-UP Misses (Market Order Alternative)

**Problem**: 1303.SR gapped past entry zone.
When a high-score stock gaps away from entry, we miss the move.

**Clarification**: For Jun 1, the "gap" was actually DOWN (16.75 entry vs 16.31 open).
The stock never hit entry because it opened LOWER.

**BUT the issue applies to true gap-ups too:**
If a stock closes at 16.75 and opens at 17.50 (+4.5% gap-up),
our entry zone (16.75-16.92) is never hit.

**Solution — Market Order for High-Confidence Picks:**
```python
# If score > 120 and predicted gap-up > 2%, use market order
if score > 120 and premarket_change > 2%:
    order_type = "MARKET"
    entry_price = "market"  # Buy at open price
    note = f"Market order — score {score}, gap-up {premarket_change:.1f}%"
```

**Risk**: Market orders are more expensive (wider spread)
**Benefit**: Don't miss the strongest movers

**Implementation**: Add to bot.py — when entry zone is far below open,
switch to market order if score > 120.
