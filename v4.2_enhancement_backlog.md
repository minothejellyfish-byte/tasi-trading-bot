## 📌 Adaptive Score Thresholds (Priority: HIGH)

### Problem (2026-06-08)
**1303.SR Case Study:**
- Score: 64.2 (below HIGH_SCORE_THRESHOLD = 80)
- Screener skipped it
- But intraday price action: +0.74% from open, hit entry zone at 16.00
- Would have been profitable if picked

**Root Cause:**
Static score threshold of 80 is too rigid for flat markets. In DEFENSIVE regime with low breadth, no stocks score above 80 even when they show good intraday movement.

### Evidence
| Date | Regime | Highest Score | Picks Generated | Missed Opportunity |
|------|--------|-------------|-----------------|------------------|
| 2026-06-08 | DEFENSIVE | 64.2 (1303.SR) | 0 | 1303.SR +0.74% |
| 2026-06-04 | DEFENSIVE | 66.3 (4191.SR) | 5 | — |
| 2026-06-03 | TRENDING | 82.1 (2084.SR) | 2 | — |

### Proposed Solution
```python
def calculate_dynamic_threshold(regime, market_breadth, avg_score):
    """Adapt score threshold based on market conditions."""
    base_threshold = 80
    
    # Regime adjustment
    regime_multipliers = {
        "TRENDING": 1.0,    # Keep at 80
        "NEUTRAL": 0.85,     # Lower to 68
        "DEFENSIVE": 0.75,   # Lower to 60
    }
    
    # Breadth adjustment (if < 30% stocks have measurable movement)
    if market_breadth < 0.3:
        breadth_adjustment = 0.9  # Reduce threshold by 10%
    else:
        breadth_adjustment = 1.0
    
    # Dynamic threshold
    threshold = base_threshold * regime_multipliers.get(regime, 1.0) * breadth_adjustment
    
    # Floor: never go below 55
    return max(threshold, 55)
```

### Implementation Requirements
1. **Market breadth calculation**: % of stocks with |gap| > 0.5% and ATR > 0.01
2. **Regime-aware scoring**: Lower threshold in DEFENSIVE/flat markets
3. **Post-hoc validation**: Track stocks that scored 55-80 but were skipped — verify if they would have been profitable

### Validation Plan
- Run adaptive thresholds on past 2 weeks of data
- Compare: static 80 vs dynamic 55-80
- Metric: Profit missed by skipped stocks in 55-80 range

### Risk Assessment
- **False positives**: Lower threshold = more picks, potentially lower quality
- **Mitigation**: Require stricter entry zones (tighter stops) when threshold is lowered
- **Fallback**: If adaptive threshold produces > 10 picks, revert to static 80

## 📌 VWAP Integration (Priority: HIGH)

### Overview
Add Volume-Weighted Average Price (VWAP) to entry zone calculation for more precise momentum measurement and better entry timing.

### Why VWAP Matters
- **Institutional Benchmark**: Algorithms and institutions use VWAP as fair value reference
- **Volume-Weighted**: Unlike simple moving averages, VWAP weights by volume — shows where real money traded
- **Dynamic Support/Resistance**: Stocks often return to VWAP during the day
- **Momentum Confirmation**: Price above VWAP = bullish momentum; below = bearish

### Current v4.1 Entry
```python
entry_low = round(min(prev_high * 0.995, close * 0.98), 2)
```

### Proposed v4.2 Entry with VWAP
```python
def calculate_entry_zone_v42(close, prev_high, prev_low, score, gap_pct=None, vwap=None):
    # Base entry from v4.1
    base_entry = min(prev_high * 0.995, close * 0.98)
    
    # VWAP enhancement (if intraday data available)
    if vwap is not None and vwap > 0:
        # Use VWAP as dynamic support
        if close > vwap:
            # Price above VWAP = bullish, tighten entry to VWAP
            base_entry = min(vwap * 0.998, base_entry)
        else:
            # Price below VWAP = bearish, use wider entry
            base_entry = min(vwap * 0.995, base_entry)
    
    # Existing gap logic from v4.1
    if gap_pct is not None:
        if gap_pct > 2.0 and score >= 120:
            return {
                'entry_low': round(close, 2),
                'entry_high': round(close, 2),
                'order_type': 'MARKET',
                'note': f'MARKET: Gap-up {gap_pct:.1f}%, score {score}'
            }
        elif gap_pct < -2.0:
            if gap_pct < -3.0 and score < 120:
                return {'skip': True, 'reason': f'Gap-down {gap_pct:.1f}% too deep'}
            base_entry = close * 0.95  # Lower entry for gap-down
    
    return {
        'entry_low': round(base_entry, 2),
        'entry_high': round(close * 1.01, 2),
        'order_type': 'LIMIT',
        'vwap_used': vwap is not None,
        'vwap': vwap
    }
```

### Implementation Requirements
1. **Data Source**: Need intraday 1-minute data from Yahoo Finance
   ```python
   df_intraday = yf.download(ticker, period="1d", interval="1m", progress=False)
   ```
2. **VWAP Calculation**: 
   ```python
   typical_price = (df['High'] + df['Low'] + df['Close']) / 3
   vwap = (typical_price * df['Volume']).cumsum() / df['Volume'].cumsum()
   ```
3. **Fallback**: If intraday unavailable, use daily VWAP approximation or skip VWAP

### When VWAP Helps Most
| Scenario | VWAP Action | Benefit |
|----------|-------------|---------|
| High-volume breakout | VWAP ≈ close | Confirms institutional participation |
| Low-volume gap | VWAP diverges from close | Detects weak momentum |
| Mid-day pullback | Price returns to VWAP | Better entry than static % |
| End-of-day hold | Price above VWAP | Confirms trend strength |

### Risk Assessment
- **Complexity**: Medium (requires intraday data fetch)
- **Latency**: +1-2 seconds per stock (intraday download)
- **False Signals**: VWAP less reliable in first 30 minutes
- **Fallback Needed**: Always have non-VWAP path if data fails

## 📌 Daily P&L Circuit Breaker (Priority: HIGH)

### Problem
Currently there is NO daily loss limit. The system can keep trading even if the entire day is going badly:
- Hard stop is per-position (-7% each stock)
- No protection for cumulative daily losses
- Could lose -5%, -10%, -15% in one day if multiple trades go wrong

### Proposed Solution
```python
DAILY_MAX_LOSS_PCT = 5.0  # Stop all trading if down 5% in one day
DAILY_SOFT_STOP_PCT = 3.0  # Warning at 3%, hard stop at 5%

def calculate_daily_pnl() -> float:
    """Calculate realized + unrealized P&L for today."""
    today_str = datetime.now(RIYADH).strftime("%Y-%m-%d")
    positions = load_positions()
    
    realized_pnl = 0
    unrealized_pnl = 0
    total_cost = 0
    
    for symbol, pos in positions.items():
        entry_cost = pos.get("total_cost", pos.get("cost", 0))
        entry_price = pos.get("entry_price", 0)
        qty = pos.get("qty", 0)
        
        if pos.get("closed"):
            # Realized: use close price
            close_price = pos.get("close_price", entry_price)
            realized_pnl += (close_price - entry_price) * qty
        else:
            # Unrealized: use current market price
            current_price = get_current_price(symbol)
            if current_price:
                unrealized_pnl += (current_price - entry_price) * qty
        
        total_cost += entry_cost
    
    total_pnl_pct = ((realized_pnl + unrealized_pnl) / total_cost * 100) if total_cost > 0 else 0
    return total_pnl_pct

def check_daily_circuit_breaker():
    """Monitor daily P&L and halt trading if limit reached."""
    daily_pnl = calculate_daily_pnl()
    
    if daily_pnl <= -DAILY_MAX_LOSS_PCT:
        # Hard stop: Halt all trading
        tg_send(f"🔴 DAILY CIRCUIT BREAKER: P&L at {daily_pnl:.1f}% (limit: -{DAILY_MAX_LOSS_PCT}%). All new trading HALTED.")
        create_stand_down("daily_circuit_breaker", "Trading halted due to daily loss limit")
        return True, "hard_stop"
    
    elif daily_pnl <= -DAILY_SOFT_STOP_PCT:
        # Soft warning: Reduce risk
        tg_send(f"⚠️ DAILY SOFT STOP: P&L at {daily_pnl:.1f}% (warning: -{DAILY_SOFT_STOP_PCT}%). Reducing position sizes by 50%.")
        return True, "soft_stop"
    
    return False, "normal"
```

### How It Works
| Level | Trigger | Action | Duration |
|-------|---------|--------|----------|
| **Normal** | P&L > -3% | Trade normally | — |
| **Soft Stop** | P&L <= -3% | Reduce size 50%, more selective | Until P&L improves |
| **Hard Stop** | P&L <= -5% | **HALT all new trades** | Rest of day |

### Implementation
- Add to poller.py: Check every 5 minutes (with slow poll)
- Add to screener.py: Check before generating new picks
- Add to bot.py: Block buy commands if circuit breaker active
- Store daily start capital at 10:00 market open
- Reset at next trading day (or manual override)

### Expected Impact
| Scenario | Without Circuit Breaker | With Circuit Breaker |
|----------|------------------------|---------------------|
| Bad day, 3 losses | -21% (3 x -7%) | -5% (halted after 2nd loss) |
| Choppy market | Keeps bleeding | Protected, live to trade tomorrow |
| Emotional trading | Revenge trades | Forced stop, prevents blowup |

### Risk Assessment
- **Complexity**: Medium (needs P&L tracking)
- **Overhead**: +1 API call per 5 min (minimal)
- **False positives**: Low (only triggers on real losses)
- **Recovery**: Manual reset next day or by command

### 5. Capital-Based Position Slots (Priority: HIGH)

#### Problem
Currently `max_positions` counts by position count, not capital used:
```python
open_count = sum(1 for p in positions.values() if not p.get("closed"))
```

This means:
- After Tier 1 partial sell (50% sold), position is still "open"
- **3 positions with partial sells = still 3/3 slots used**
- Capital freed from partial sell **cannot** be used for new positions
- System misses opportunities even when capital is available

#### Example
| Scenario | Positions | Capital Used | Old Logic | New Logic |
|----------|-----------|--------------|-----------|-----------|
| 3 full positions | 3 open | 100% | 3/3 full | 3/3 full |
| Tier 1 on 1 position | 3 open | 82.5% | **3/3 full** | **2.5/3 → 0.5 slot free!** |

#### Proposed Solution
```python
def calculate_open_slots(positions, total_capital, regime):
    """Calculate open position slots based on capital allocation."""
    position_pct = regime.get("position_pct", 0.35)
    max_slots = regime.get("max_positions", 3)
    slot_size = total_capital * position_pct
    
    # Sum invested capital in open positions
    used_capital = sum(
        p.get("total_cost", p.get("cost", 0))
        for p in positions.values()
        if not p.get("closed")
    )
    
    used_slots = used_capital / slot_size
    open_slots = max(0, max_slots - used_slots)
    
    return open_slots

# Usage:
open_slots = calculate_open_slots(positions, total_capital, regime)
if open_slots >= 1.0:
    # Full slot available → standard position
    buy_qty = calculate_position_size(total_capital, 1.0)
elif open_slots >= 0.5:
    # Partial slot → smaller position
    buy_qty = calculate_position_size(total_capital, open_slots)
    tg_send(f"⚡ Partial slot: Opening {open_slots:.0%} size position")
```

#### Benefits
| Feature | Impact |
|---------|--------|
| **Capital efficiency** | Freed capital from partial sells gets reused |
| **More opportunities** | Can open smaller positions when slots partially free |
| **Flexibility** | Position size scales with available capital |
| **No wasted capital** | Money from Tier 1 sells goes to work immediately |

- Modify screener.py entry logic for partial slot sizing

### 6. Liquidity Filter — Minimum WS Tick Frequency (Priority: HIGH)

**Problem (2026-06-08, 9409.SR)**
- Stock picked by mid-screen with +6.49% change and score 70.2
- But only **9 WS ticks in 40+ minutes** (10:02 → 10:37)
- After 10:06, no real trades — only stale repeated frames with same price (10.00)
- yfinance confirms: only 2 bars for entire session, stock essentially frozen
- Poller correctly rejected stale WS cache, fell back to yfinance delayed price
- Result: Pick generated but stock is illiquid and untradeable

**Root Cause**
The screener uses websocket bars to calculate intraday momentum, but doesn't verify that the stock has sufficient real-time trading activity. A stock can show high %change from a few early trades then go completely silent.

**Evidence**
| Symbol | WS Ticks (10:00-10:40) | Last Real Trade | Pickable? |
|--------|------------------------|-----------------|-----------|
| 9409.SR | 9 | 10:06 | ❌ No — illiquid |
| 1304.SR | 200+ | Continuous | ✅ Yes |
| 2110.SR | 150+ | Continuous | ✅ Yes |

**Proposed Solution**
```python
MIN_WS_TICKS_PER_HOUR = 30  # Require at least 30 real ticks for liquidity
MIN_REAL_TRADES_PER_HOUR = 5  # Require at least 5 trades with actual price changes

def check_liquidity(ws_data: dict, symbol: str, window_minutes: int = 60) -> bool:
    """Check if stock has sufficient real-time trading activity."""
    ticks = ws_data.get(symbol, [])
    
    if len(ticks) < MIN_WS_TICKS_PER_HOUR * (window_minutes / 60):
        return False, f"Only {len(ticks)} ticks in {window_minutes}min (need {MIN_WS_TICKS_PER_HOUR})"
    
    # Count trades with actual price changes (real=True in ws_logger)
    real_trades = sum(1 for t in ticks if t.get("real", False))
    if real_trades < MIN_REAL_TRADES_PER_HOUR:
        return False, f"Only {real_trades} real trades (need {MIN_REAL_TRADES_PER_HOUR})"
    
    # Check for stale prices (same price repeated >5 times)
    prices = [t["price"] for t in ticks]
    price_changes = sum(1 for i in range(1, len(prices)) if prices[i] != prices[i-1])
    if price_changes < 3:
        return False, f"Price frozen (only {price_changes} changes)"
    
    return True, "Liquid"
```

**Integration Points**
- Add to `midscreen_ws.py`: Reject bars from illiquid stocks before scoring
- Add to `screener.py`: Verify pre-market liquidity before picking
- Add to `poller.py`: Skip entry signals for stocks with stale WS cache

**Expected Impact**
| Metric | Without Filter | With Filter |
|--------|---------------|-------------|
| Illiquid picks | ~15% of picks | 0% |
| Entry slippage | High (no market depth) | Normal |
| False signals | From frozen prices | Eliminated |
| Missed opportunities | Some valid low-volume stocks | Minimal |

**Risk Assessment**
- **False negatives**: Might skip stocks with genuine low-volume breakouts
- **Mitigation**: Make thresholds configurable per regime (DEFENSIVE: stricter, TRENDING: looser)
- **Validation**: Track rejected stocks for 2 weeks to verify none were profitable

### Proposed Design

**`orders.json` schema (new file):**
```json
{
  "12345": {
    "initiated_at": "2026-06-10T13:32:27+03:00",
    "initiated_by": "auto_buy" | "auto_sell" | "manual-buy" | "manual-sell",
    "symbol": "2200",
    "side": "BUY" | "SELL",
    "qty": 25,
    "price": 7.66,
    "type": "MARKET" | "LIMIT",
    "status": "INITIATED",
    "status_history": [
      {"status": "INITIATED", "at": "...", "by": "poller"},
      {"status": "PLACED",    "at": "...", "by": "bookkeeper"}
    ]
  }
}
```

**Status Lifecycle:**
- `INITIATED` → poller/bot writes immediately on place_order success
- `PLACED` → bookkeeper sees it in Order/List (Derayah accepted it)
- `FILLED` → bookkeeper sees filled status → moves to positions.json
- `REJECTED` → bookkeeper sees it never made it to Order/List (Derayah rejected on place)
- `CANCELLED` → bookkeeper sees cancelled status
- `EXPIRED` → order disappeared from Order/List (likely end-of-day cleanup)

**Writer Pattern (same as positions.json/capital.json):**
```
POLLER (immediate write, source = "auto-buy"):
  1. POST /trading/Order/Place → success
  2. WRITE orders.json[order_id] = {status: INITIATED, ...}
  3. EXISTING: write positions.json (if BUY and market filled)
  4. EXISTING: update capital.json
  5. NEW: trigger_bookkeeper_sync() (just like bot does)
  6. ANNOUNCE: "📤 INITIATED: BUY 2200 25 @ 7.66 LIMIT (orderId=12345)"

BOOKKEEPER (5-min overwrite, source = "derayah-overwrite"):
  1. POST /trading/Order/List → all orders
  2. WRITE orders.json from scratch (overwrite pattern)
  3. For each order: set status from API, append to status_history
  4. Compute deltas vs previous orders.json:
     - INITIATED → PLACED: ANNOUNCE "✅ PLACED: order 12345 confirmed by Derayah"
     - INITIATED → REJECTED: ANNOUNCE "❌ REJECTED: order 12345 (reason)"
     - PLACED → FILLED: ANNOUNCE "🎯 FILLED: order 12345 — 25×2200 @ 7.66"
     - PLACED → CANCELLED: ANNOUNCE "❌ CANCELLED: order 12345"
     - Disappeared from API: ANNOUNCE "⚠️ order 12345 not in API — assuming EXPIRED"
  5. Update capital.json.booked based on sum of (qty × price) for INITIATED/PLACED orders
```

**`capital.json` restructure (3 buckets):**
```json
{
  "equity": 12345.67,            // current market value of positions
  "booked": 1500.00,             // sum of (qty × price) for INITIATED/PLACED orders
  "cash": 5000.00,               // available to invest/withdraw
  "available_capital": 5000.00,  // legacy field (= cash) for backward compat
  "total_fees": 12.50,
  "updated_at": "...",
  "source": "auto-buy" | "bookkeeper-overwrite" | "derayah-dashboard-scraped-refresh"
}
```

**Where capital values come from (per A A's guidance 2026-06-10 23:33):**
- `equity` → computed locally: sum of (qty × current_price) for all positions
- `booked` → computed locally: sum of (qty × price) for INITIATED/PLACED orders
- `cash` → scraped from Derayah dashboard (already done in `bot.py:get_actual_balance_from_derayah`)
- **Concern:** Dashboard navigation takes ~11 seconds (page.reload + 3s wait + page.goto + 8s wait). This is the same navigation we use for session keepalive. May be acceptable for 5-min cadence.

### Local Pre-Check in Poller (per A A 2026-06-10 23:33)

**Double-sell guard:**
```python
def effective_holdings(symbol: str) -> int:
    """Compute holdings as filled - outstanding_sell + outstanding_buy."""
    positions = load_positions()
    filled_qty = positions.get(symbol, {}).get("qty", 0)
    
    orders = load_orders()
    outstanding_buy = sum(o["qty"] for o in orders.values() 
                         if o["symbol"] == symbol and o["side"] == "BUY" 
                         and o["status"] in ("INITIATED", "PLACED"))
    outstanding_sell = sum(o["qty"] for o in orders.values() 
                          if o["symbol"] == symbol and o["side"] == "SELL" 
                          and o["status"] in ("INITIATED", "PLACED"))
    return filled_qty - outstanding_sell + outstanding_buy
```

**Capital availability check (per A A 2026-06-10 23:40):**
- Poller MUST block order if `cash < total_cost` (qty × price + fees)
- A A explicitly added this to the v1 implementation (was previously in deferred)
- Reason: prevents placing orders you can't actually fund
- Compute as: `total_cost = qty × price + (qty × price × 0.0005) + (qty × price × 0.0005 × 0.15)`
  - Commission: 0.05% of trade value
  - VAT: 15% of commission
- Read `cash` from capital.json (bookkeeper keeps it fresh from Derayah scrape)
- If blocked: ANNOUNCE `❌ BLOCKED: insufficient cash (need 1500.50, have 1200.00) — order not placed`

### Bookkeeper Reconciliation Logic (updated)

```python
def reconcile_orders():
    """Overwrite orders.json with Derayah truth, announce changes."""
    api_orders = get_orders_api()  # POST /trading/Order/List
    api_order_map = {str(o.get("orderId")): o for o in api_orders}
    
    # Load current local state
    local_orders = load_orders()
    
    # For each API order: write to orders.json with mapped status
    new_orders = {}
    for oid, api_o in api_order_map.items():
        derayah_status = api_o.get("orderStatusId") or api_o.get("status")
        our_status = map_derayah_status(derayah_status)
        
        # Preserve original initiated_at/initiated_by if exists
        existing = local_orders.get(oid, {})
        new_orders[oid] = {
            "initiated_at": existing.get("initiated_at") or (api_o.get("orderDateTime") or _now()),
            "initiated_by": existing.get("initiated_by") or "derayah-direct",
            "symbol": api_o.get("symbol", ""),
            "side": "BUY" if api_o.get("side") == 1 else "SELL",
            "qty": api_o.get("quantity", 0),
            "price": api_o.get("price", 0),
            "type": "MARKET" if api_o.get("price", 0) == 0 else "LIMIT",
            "status": our_status,
            "updated_at": _now(),
        }
    
    # For each local order NOT in API: mark appropriately
    for oid, local_o in local_orders.items():
        if oid not in api_order_map:
            if local_o["status"] in ("FILLED", "REJECTED", "CANCELLED", "EXPIRED"):
                # Already terminal, drop from orders.json (will go to Google Sheet)
                continue
            elif local_o["status"] == "INITIATED":
                # Still in INITIATED — Derayah doesn't have it
                # Mark as REJECTED (Derayah never accepted it)
                new_orders[oid] = {**local_o, "status": "REJECTED", "updated_at": _now()}
                # ANNOUNCE: "❌ REJECTED: order 12345 (Derayah didn't accept within 5 min)"
            elif local_o["status"] == "PLACED":
                # Was placed, now gone — probably expired/cancelled
                new_orders[oid] = {**local_o, "status": "EXPIRED", "updated_at": _now()}
                # ANNOUNCE: "⚠️ EXPIRED: order 12345 (no longer in Derayah)"
    
    # Save (overwrite pattern, same as positions.json)
    save_orders(new_orders)
    
    # Announce deltas
    announce_order_deltas(local_orders, new_orders)
```
